## 논문 진행
논문은 매주 교수님의 요청에 따른 업무를 수행했다.
1. SOL, AVAIL 데이터 다시 추출
  전체 기간동안에 데이터를 가지고 logistic 분석을 진행하고 싶어서 추출을 다시 진행했다.
  SOL에 경우 전체 데이터를 뽑는 것이 여건상 불가능하고, AWS를 이용하기엔 경험이 없고 비용적인 측면에서 포기했다. 그래서 Bitquery로 amount>=100인 transfer 데이터만 추출했다. 
  AVAIL의 경우 해당 API를 이용해서 블록을 추출한 후에 거래내역만 뽑아서 정리했다. 가격은 bitumb데이터의 분단위가 제일 필요하다고 생각되었지만 얻을 방법을 못찾았다.
2. 해당 자료로 network 지표를 계산
  전과 같은 방법으로 계산하돼, 가중치를 부여한 경우도 계산을 진행했다. 데이터의 크기가 크기때문에 상당한 시간이 소요되었다.
3. supersinker, supersource list에 코인 보유량 및 현재 top holders list -> AVAIL은 기간이 너무 짧아서 조사는 했으나 뒤에 사용하진 않은 것 같다.
  각 시간별대로 super에 해당하는 주소의 보유량을 알 수 있음 좋겠지만 해당 자료를 모두 추적 조사하는 것에는 현실적인 한계가 존재하여 특정 일자의 snap shot을 기준으로 조사를 했다.
  WEMIX와 AVAIL은 해당 scan에서 holder들의 리스트를 볼 수 있어서 쉽게 조사가 가능했다.
  SOL에 경우 어떤 scan 사이트에서도 제공해주고 있지 않아 여러 조사 끝에 dune 사이트를 이용하여 조사했다.
  그리고 super에 해당하는 주소에 코인 보유량은 bitquery와 scan사이트에서 조회하서 정리했다. 그리고 super 주소가 top holder에 들어가는지 랭킹을 매치했다.
4. 계획이 변경되어 추가적인 기술 탐색은 멈췄다.
  그래서 논문 조사만을 조사하고 마무리되었다.
