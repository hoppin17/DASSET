# DASSET

이벤트 기반 온체인 네트워크 분석과 주요 지갑 추적

이 저장소는 2024년 서울대학교 빅데이터 핀테크 8기 기업실습에서 수행한 캡스톤 프로젝트와 이후의 논문 작성 보조 작업을 정리한 연구 아카이브입니다. 특정 이벤트 전후에 온체인 네트워크가 어떻게 달라지는지 살펴보고, 변화에 관여한 주요 지갑을 추적하는 과정을 담고 있습니다.

> 이 프로젝트는 완성된 이상거래 탐지 서비스가 아닙니다. 공개 온체인 데이터로 관찰 가능한 신호와 분석 방법을 탐색한 연구 기록입니다.

## 왜 시작했나

프로젝트가 진행된 2024년에는 가상자산 이용자 보호 제도가 시행되기 시작했지만, 거래소 밖에서 발생하는 온체인 활동을 어떻게 관찰하고 해석할지에 관한 연구는 아직 충분하지 않았습니다.

블록체인 거래는 공개되어 있지만 주소만으로 지갑의 주체를 확정하기 어렵고, 데이터의 양도 매우 큽니다. 하나의 거래를 내부자 거래나 이상거래라고 바로 판단할 수도 없습니다. 이 프로젝트는 다음 질문에서 출발했습니다.

- 상장, 상장폐지, 락업 해제, 토큰 소각 같은 이벤트 전후에 네트워크 구조가 달라지는가?
- 네트워크 지표의 변화로 평소와 다른 시점을 찾을 수 있는가?
- 거래량이나 거래빈도가 두드러진 지갑을 추려 그 역할을 추정할 수 있는가?

## 어떻게 분석했나

가격·이벤트 수집 → 온체인 데이터 수집 → 이벤트 구간 설정 → 네트워크 지표 계산 → 동적 시각화 → 주요 지갑 라벨링·거래 흐름 추적

중간발표 단계에서는 12개 가상자산의 가격과 이벤트를 조사하고, AVAX와 PLA/PDA를 중심으로 시계열 분해와 Granger 인과관계 분석을 시도했습니다. 이후에는 복잡한 온체인 변화를 더 직관적으로 보여주기 위해 네트워크 분석에 집중했습니다.

기말발표에서는 WEMIX의 상장폐지, 거래지원 종료, 소각, 재상장 전후에 그린 네트워크 그래프에서 평소와 다른 양상을 발견했고, 해당 구간의 변화를 중심으로 지표와 주요 지갑을 추가로 살펴봤습니다. 지갑을 노드, 토큰 이동을 방향이 있는 엣지로 보고 다음 지표와 패턴을 일별로 비교했습니다.

- Degree, Betweenness, Closeness Centrality와 PageRank
- Reciprocity, Assortativity, Transitivity, Clustering Coefficient
- 네트워크 밀도와 연결요소
- Indegree·Outdegree 비율의 누적분포(CDF)
- 상위 Sender·Receiver, 거래금액·거래빈도, Self-loop

저장소에 남아 있는 대표 네트워크 시각화 구간은 `2022.11.10~11.24`, `2022.11.25~12.10`, `2023.02.09~02.23`입니다. 각 구간은 주요 이벤트의 앞뒤 모습을 비교할 수 있도록 설정했습니다.

### 살펴본 내용

- 상장폐지 결정, 거래지원 종료, 재상장 같은 이벤트의 앞뒤에서 네트워크 그래프의 노드 집중도와 연결 양상이 평소와 다르게 보이는 시점이 있었습니다.
- 이 특이점을 조금 더 구체적으로 살펴보기 위해 평균 중심성, 네트워크 밀도, Indegree·Outdegree 분포 등의 변화를 함께 비교했습니다.
- 하나의 지표만으로 판단하기보다 여러 지표와 그래프의 변화 양상을 함께 보는 방식에 주안점을 두었습니다.
- 상위 Sender·Receiver와 거래금액 상위 지갑을 추린 뒤 거래소, 재단, 스마트 컨트랙트, 개인 지갑 후보로 라벨링하고 거래 흐름을 추적했습니다.
- 반복적인 Self-loop와 짧은 시간 안에 여러 지갑을 거치는 이동 등 추가 조사가 필요한 패턴도 확인했습니다.

이 관찰만으로 이벤트와 거래의 인과관계나 지갑 주체의 위법성을 확정할 수는 없습니다. 실제 분석에서도 내부자 지갑을 특정하지 못했으며, 스왑 이후의 흐름과 라벨이 없는 지갑을 추적하는 데에는 분명한 한계가 있었습니다.

## 논문 작성 보조

캡스톤 이후인 2024년 9월부터 10월까지 약 2개월간 논문 작성을 위한 데이터 작업을 보조했습니다. 논문의 주된 집필을 맡은 것은 아니며, WEMIX, SOL, AVAIL 데이터의 수집·정제와 분석에 필요한 기술 작업을 지원했습니다.

- SOL은 전체 데이터를 수집하기 어려워 `amount >= 100`인 트랜스퍼를 중심으로 분석했습니다.
- 일별 중앙성, 가중 중앙성, 연결요소와 연결성 지표를 계산했습니다.
- 주요 지갑과 상위 보유자 정보를 결합하고 지갑별 송수신 흐름을 정리했습니다.
- AVAIL은 Subscan API에서 블록을 수집한 뒤 이벤트 데이터로 변환하는 방식을 시험했습니다.

이 작업은 상사법학회에 제출·게재된 [**가상자산 온체인 데이터 분석을 통한내부자 거래 규제 가능성에 대한 연구**](https://www.kci.go.kr/kciportal/ci/sereArticleSearch/ciSereArtiView.kci?sereArticleSearchBean.artiId=ART003144407)의 데이터 분석 보조로 수행했습니다. 현재 저장소에는 논문 작성을 지원하며 진행한 데이터 수집·정제, 네트워크 지표 계산, 주요 지갑 탐색 과정이 남아 있습니다.

## 데이터와 재현 범위

기업 요청과 파일 크기 문제로 원본·중간 데이터는 저장소에 포함하지 않았습니다. 따라서 현재 저장소만으로 모든 분석을 처음부터 그대로 재현할 수는 없습니다.

노트북의 경로는 저장소 기준 상대경로로 정리되어 있으며, 로컬 데이터는 `data/raw`, `data/interim`, `data/processed` 아래에 배치합니다. 데이터 수집 노트북은 API 키를 코드에 저장하지 않고 `BITQUERY_API_KEY`, `TONCENTER_API_KEY` 환경변수에서 읽습니다. 자세한 규칙은 [data/README.md](data/README.md)를 참고하세요.

## 해석 시 유의사항

- 온체인 주소 라벨은 블록 탐색기와 공개 서비스에 기반한 추정이며 실제 소유자를 보장하지 않습니다.
- 이벤트와 지표 변화가 같은 시기에 나타났다는 사실은 인과관계를 뜻하지 않습니다.
- 데이터 수집 범위와 필터 조건에 따라 네트워크 구조와 상위 지갑 결과가 달라질 수 있습니다.
- 이 저장소의 결과는 연구·교육 목적의 탐색 결과이며 투자 또는 법률 판단의 근거가 아닙니다.
