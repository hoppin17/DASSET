# 논문 작성 보조 기록

## 개요

- 기간: 2024년 9월~10월, 약 2개월
- 역할: 논문 집필이 아닌 데이터 수집·정제, 네트워크 지표 계산, 주요 지갑 탐색 보조
- 관련 논문: [**가상자산 온체인 데이터 분석을 통한내부자 거래 규제 가능성에 대한 연구**](https://www.kci.go.kr/kciportal/ci/sereArticleSearch/ciSereArtiView.kci?sereArticleSearchBean.artiId=ART003144407)

교수님의 요청에 따라 기존 팀원들과 함께 논문 작성에 필요한 데이터 작업을 보조했습니다.

## 수행 내용

### 1. SOL·AVAIL 데이터 추가 수집 및 정제

논문 분석에 활용할 전체 기간 데이터를 마련하기 위해 SOL과 AVAIL 데이터를 추가로 수집했습니다.

SOL 전체 전송 데이터를 수집하는 것은 데이터 규모와 비용 문제로 어려웠습니다. AWS를 이용하는 방법도 검토했지만 당시 경험과 비용 여건을 고려해 사용하지 않았습니다. 대신 Bitquery에서 거래량(`amount`)이 100 이상인 전송 데이터만 수집했습니다.

AVAIL은 Subscan API에서 블록 데이터를 수집한 뒤 전송 관련 이벤트를 추출해 정리했습니다. 가격 데이터는 빗썸의 분 단위 자료가 필요하다고 판단했지만, 이를 확보할 방법을 찾지 못했습니다.

### 2. 네트워크 지표 계산

수집한 데이터를 이용해 기존 캡스톤과 같은 방식으로 네트워크 지표를 계산했습니다. 거래량에 가중치를 적용한 지표도 함께 계산했으며, 데이터 규모가 커 처리에 상당한 시간이 소요됐습니다.

### 3. 주요 지갑과 상위 보유자 비교

분석에서 super-sink와 super-source로 분류된 주소의 코인 보유량과 상위 보유자(top holders) 포함 여부를 조사했습니다.

주소별 보유량을 시점마다 모두 추적하는 것은 현실적으로 어려워 특정 날짜의 스냅숏(snapshot)을 기준으로 비교했습니다. WEMIX와 AVAIL은 각 블록 탐색기에서 보유자 목록을 확인할 수 있었습니다. SOL은 블록 탐색기에서 필요한 목록을 찾기 어려워 조사 끝에 Dune을 이용했습니다.

이후 Bitquery와 블록 탐색기에서 해당 주소의 보유량을 확인하고, 각 주소가 상위 보유자 목록에 포함되는지 대조했습니다. AVAIL은 분석 기간이 짧아 관련 조사의 활용 범위가 제한적이었습니다.

### 4. 데이터 수정·전달 및 마무리

앞선 분석 결과를 바탕으로 교수님과 변호사님이 논문 작성에 필요한 추가 데이터 정제를 요청했습니다. 요청받은 방향에 맞춰 데이터를 수정·정리한 뒤 전달했으며, 이를 끝으로 논문 작성 보조 작업을 마무리했습니다.
